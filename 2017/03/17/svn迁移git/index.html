svn迁移git

发表于 2017-03-17   |   分类于 教程   |  

0：程序员从0开始

刚开始公司让转的时候其实我是拒绝的，因为你不能让我转就转，我总得试一下。然后发现Win上svn转git是真TMTMTM的方便，因为只需要安装一个小乌龟git版，全程无痛可视化操作。无奈用了MAC，只能继续寻找可用的方法，各种百度谷歌尝试过后，终于迁移成功。（windows也可用如下方式）

既然准备从svn迁移git，首先本地得安装git客户端。
安装好后执行如下命令

1：生成密钥

ssh-keygen -t rsa -C "kylinz007"
执行此命令生成一对RSA密钥,引号中填入github注册邮箱账号或用户名

cat ~/.ssh/id_rsa.pub
复制显示出来的信息，注意一定要复制完整

2：在你的git账号上添加密钥，创建仓库

登录进入github，点击右上角头像，选择settings，点击左边菜单栏中的SSH and GPG keys，右上角有个new SSH key，将复制好的公钥粘贴到key栏位中，title可以任意输入，然后保存。

在Repositories中new一个新仓库

3：本地配置github

创建一个目录作为git的根目录
cd命令进入该目录

将svn上的项目先下载到本地（包含主干，分支，标记，所有提交记录）
git svn clone 'svn项目路径' -t tags -b branches -T trunk 项目名

将.git/refs/remotes目录相应分支文件COPY至.git/ref/heads目录下
cd 项目名/.git/refs/remotes
mv origin ../heads/

4：推送至git公共仓库

配置git账户地址信息:
git config --global user.name "kylinz007"
git config --global user.email "kylinz007@163.com"
git remote add origin git@github.com:kylinz007/git2svn

push至Github仓库
git push -u origin –all

大功告成！
